### Proyecto: Creación de un Rubber Ducky para prácticas de ciberseguridad

En este proyecto hemos llevado a cabo la creación de un dispositivo tipo Rubber Ducky, una herramienta utilizada en el ámbito de la ciberseguridad para la automatización de comandos mediante la emulación de un teclado USB mediante una conexión bluethoot. 
Este tipo de dispositivos permite ejecutar scripts de forma rápida en un equipo objetivo, lo que resulta útil tanto para pruebas de penetración como para la concienciación en seguridad informática.
Para el desarrollo del proyecto, seguimos un tutorial en el que se explicaba paso a paso cómo configurar el dispositivo, cargar los scripts y verificar su funcionamiento.
A lo largo del proceso, aprendimos a programar instrucciones en lenguaje específico (ArduinoScript), a convertirlas en un formato ejecutable por el dispositivo y a transferirlas correctamente.
Una vez finalizado el montaje y la configuración, logramos que el Rubber Ducky ejecutara correctamente los comandos previstos al conectarlo a un ordenador(abriendo la CMD), cumpliendo así con los objetivos planteados. 
Este resultado demuestra tanto la viabilidad del proyecto como la importancia de comprender este tipo de herramientas dentro del campo de la ciberseguridad.
En conclusión, este proyecto nos ha permitido adquirir conocimientos prácticos sobre automatización, seguridad informática y posibles vulnerabilidades en sistemas, 
destacando la importancia de implementar medidas de protección frente a este tipo de ataques.

### Project: Creating a Rubber Ducky for Cybersecurity Practices

In this project, we developed a *Rubber Ducky*-type device, a tool used in the field of cybersecurity for automating commands by emulating a USB keyboard through a Bluetooth connection.
These types of devices allow scripts to be executed quickly on a target computer, making them useful for both penetration testing and cybersecurity awareness.
For the development of the project, we followed a tutorial that explained step by step how to configure the device, load the scripts, and verify its functionality.
Throughout the process, we learned how to program instructions in a specific language (ArduinoScript), convert them into a format executable by the device, and transfer them correctly.
Once the assembly and configuration were completed, we successfully got the Rubber Ducky to execute the intended commands when connected to a computer (opening the Command Prompt), thus achieving the proposed objectives.
This result demonstrates both the feasibility of the project and the importance of understanding these types of tools within the field of cybersecurity.
In conclusion, this project allowed us to acquire practical knowledge about automation, information security, and potential system vulnerabilities, highlighting the importance of implementing protective measures against such attacks.

在这一过程中，我们学习了如何使用特定语言（ArduinoScript）编写指令，将其转换为设备可执行的格式，并正确地进行传输。

在完成组装和配置后，我们成功实现了 Rubber Ducky 在连接到计算机时执行预设命令（打开命令提示符），从而达成了既定目标。
这一结果不仅证明了项目的可行性，也体现了理解此类工具在网络安全领域中的重要性。

总的来说，本项目使我们获得了关于自动化、信息安全以及系统潜在漏洞的实践经验，同时也强调了针对这类攻击实施防护措施的重要性。
